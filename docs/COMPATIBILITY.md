# 兼容性与功能降级

## 发布包没有包含什么

为了避免重新分发易优 CMS 核心、覆盖不同版本源码或泄露配置，本包不包含原站的：

- `application` 控制器、模型、后台模板和路由修改；
- `core` 标签库修改；
- `application/database.php` 与任何本地配置；
- CMS 安装程序、完整数据库或运行目录；
- 原站广告、商务联系内容和私有运维脚本。

目标站必须先自行合法安装易优 CMS。本包是模板覆盖层、公开数据和可选增强说明，不是完整 CMS 发行版。

## 已由导入器补齐的依赖

- 产品内容表 7 个场景排序字段；
- `wogs_tool_category_rel` 多分类关系表；
- 原栏目 ID 和文档 AID；
- 标签、缩略图和公开内容；
- 模板所需的自定义 HTML/CSS/JS/图片文件。

## 会自动降级的功能

- 多分类关系表不存在时，模板多数查询会退回主分类 `archives.typeid`；
- 后台没有广告记录时，广告组件为空或显示通用占位图；
- 没有联系方式全局字段时，页脚相关入口为空，站长应在后台重新配置；
- 没有推荐内容时，推荐榜为空或使用模板中的普通热门回退；
- 排行页路由不可用时，可先使用“热点排行”栏目的默认动态 URL。
- 开源包已移除许可证不允许用于 Open Source 的 Fancybox 6.1.14，并改用 MIT 许可的 PhotoSwipe 5.4.4；正文图片支持弹层浏览、缩放、键盘切换和移动端手势。若目标站启用了严格 CSP，请允许模板自身的静态 CSS/JS，并避免拦截 `/template/pc/skin/Lib/photoswipe/`。

## 需要按目标版本适配的功能

### 后台扩展字段 UI

导入数据可以直接使用场景置顶值，但新建/编辑工具时，目标后台默认不显示这些扩展字段。让 AI 根据 [BACKEND_RANKING_GUIDE.md](BACKEND_RANKING_GUIDE.md) 做最小增量修改，或先使用 `set_tool_ranking.php`。

### URL 路由与伪静态

模板使用 `/ranking/`、`/submit-tool/`、`/business/` 等友好路径。不同易优版本和 Web 服务器的伪静态规则不同，应优先通过栏目目录和后台 URL 模式解决，不要直接覆盖整个路由文件。

### 搜索与相关内容

原开发站存在搜索相关性和部分控制器优化，但这些核心改动没有进入开源包。全新站仍可使用易优 CMS 标准搜索；结果排序和原站可能不同。

### 管理后台批量操作

原站可能有批量场景排序、内容安全检查等后台定制。本包只保证前台模板和数据导入，不承诺这些后台定制存在。

## 版本适配原则

让 AI 适配时应先读取目标版本文件，再做小范围修改：

1. 不拿其他版本的 `application` 或 `core` 文件整份覆盖；
2. 不修改数据库连接文件；
3. 新字段使用存在性检查，重复执行不报错；
4. 模板函数加 `function_exists` 防止重复声明；
5. 自定义表名必须使用目标配置的表前缀；
6. 每次修改后清缓存并分别测试首页、列表、详情、搜索和排行；
7. 保持 UTF-8，发现乱码先验证文件字节和浏览器输出。

## 栏目编号说明

本包为了兼容现有模板逻辑保留源栏目 ID 和文档 AID。因此它适合全新 CMS。若必须导入已有站，正确做法是：

1. 建立旧 ID → 新 ID 映射；
2. 重写 JSON 中主分类、多分类和父子关系；
3. 搜索模板中的硬编码栏目 ID 并按映射替换；
4. 改造导入器使用新 ID；
5. 在副本数据库完整演练后再用于生产站。

不要关闭冲突检查直接导入。
